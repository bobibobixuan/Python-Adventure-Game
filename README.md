# Python基础闯关游戏

一个基于原生 HTML、CSS、JavaScript 开发的 Python 基础闯关小游戏。项目当前包含两个学习单元，支持冒险模式、极限测试、修炼场、做题记录、错题分析和成就系统。

## 项目特点

- 原生前端实现，无需额外框架
- 单页游戏界面，适合本地直接打开使用
- 两个教学单元，共 12 个关卡
- 支持选择题、判断题、填空题三种题型
- 支持关卡星级、连击、积分、生命值与计时机制
- 支持修炼场随机练习
- 支持极限测试和双单元综合挑战
- 使用 localStorage 保存学习进度、成就和错题记录

## 当前内容

### 单元设置

1. 第一单元：运算符进阶
2. 第二单元：If语句基础

### 模式说明

1. 冒险模式：按单元逐关解锁，完成关卡可获得星级。
2. 极限测试：一命到底，答错即终止。
3. 修炼场：不计时、不扣血，适合反复练习。
4. 做题记录：查看累计题量、正确率、星级与极限测试成绩。
5. 错误分析：查看历史错题与知识点解析。
6. 成就一览：根据答题表现解锁成就。

## 目录结构

```text
.
├─ build.py
├─ dev
│  ├─ data.js
│  ├─ game.js
│  ├─ index.html
│  └─ style.css
└─ dist
```

## 文件说明

- dev/index.html：页面骨架，包含各个界面的 HTML 结构以及打包占位符。
- dev/style.css：游戏的样式、动画和响应式布局。
- dev/data.js：题库数据、单元配置、成就配置。
- dev/game.js：游戏状态、答题流程、界面切换、记录与成就逻辑。
- build.py：将开发目录中的 HTML、CSS、JS 合并为单文件版本。

## 本地使用

### 方式一：直接打开开发版页面

直接在浏览器中打开 dev/index.html 即可。

说明：
index.html 依赖其中的打包占位符注入脚本和样式，因此如果你要运行完整内容，通常建议先执行打包，或者确保开发版文件已经按正确方式引入。

### 方式二：打包后打开单文件版本

在项目根目录运行：

```bash
python build.py
```

执行后会生成：

```text
dist/Python基础闯关_正式版.html
```

然后直接用浏览器打开这个 HTML 文件即可。

## 打包机制

build.py 会执行以下操作：

1. 读取 dev/index.html
2. 读取 dev/style.css，并包装为 style 标签
3. 读取 dev/data.js，并包装为 script 标签
4. 读取 dev/game.js，并包装为 script 标签
5. 将占位符替换到 HTML 中
6. 输出到 dist 目录

其中 index.html 中使用了以下占位符：

- <!-- BUILD:STYLES -->
- <!-- BUILD:DATA -->
- <!-- BUILD:GAME -->

## 数据与存档

游戏使用浏览器 localStorage 保存以下内容：

- 关卡解锁状态
- 关卡星级
- 成就列表
- 错题记录
- 累计答题数与正确数
- 修炼场次数
- 极限测试通关次数

如果需要重置记录，可以使用游戏内的记录重置功能，或手动清除浏览器本地存储。

## 适合的维护方式

如果后续要继续扩展内容，建议按下面方式维护：

1. 新增题目时，优先修改 dev/data.js。
2. 新增界面或调整交互时，修改 dev/index.html 和 dev/game.js。
3. 调整视觉样式时，修改 dev/style.css。
4. 每次改动完成后执行 python build.py 重新生成发布文件。
